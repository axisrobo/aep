# Harmovela Protocol 路线图

> 协议的正式工作名称是 **Harmovela Protocol**。当前 0.1 草案仍是兼容性基线，公共标识和技术标识符将通过版本化发布逐步迁移。旧有技术标识符（如 `schemas/aep-envelope.schema.json` 和 `AEP-C3`）在迁移完成前保持有效。

## 发展方向

Harmovela 的价值通过 **能力成熟度栈** 逐步积累。每一层依赖于下一层，只有在底层稳定后才具备意义。事件是基础；协调是当前前沿；适配是远景。

| 层 | 回答的问题 | 自主级别 | 状态 | 详情 |
| --- | --- | --- | --- | --- |
| **事件 (Event)** | 发生了什么？——类型化、可关联的通信基础 | L0 | 已交付/稳定 | [event-layer.md](roadmap/event-layer.md) |
| **协调 (Coordination)** | 谁做什么，基于什么共享事实？——委派、所有权、状态一致 | L1–L2 | 活跃推进，语义收紧中 | [coordination-layer.md](roadmap/coordination-layer.md) |
| **适配 (Adaptation)** | 系统如何观察、调整和进化？——反馈、产品化自主边界 | L3（在范围内）；L4/AGI（非目标） | 未来 | [adaptation-layer.md](roadmap/adaptation-layer.md) |

**当前状态：** 事件层已完成且稳定。当前活跃工作是在收紧协调层，使多个智能体能够被委派并以可互操作的方式协调共享状态。适配层——反馈，然后是受治理的产品化自主能力——是下一个远景。

## 自主能力阶梯

Harmovela 是一个协调协议，而非智能模型。版本里程碑仅承诺每一级所需的协调语义，绝不承诺实现智能体的智能程度。

| 级别 | 定义 | 状态 | 层 |
| --- | --- | --- | --- |
| L0 | 事件感知智能体 | 已支持 | 事件 |
| L1 | 有界自主任务智能体 | 已支持，但依赖实现者策略 | 协调 |
| L2 | 多智能体协作和委派 | 部分支持；语义需要收紧 | 协调 |
| L3 | 具有审计、预算和授权边界的产品化自主能力 | 尚未实现 | 适配 |
| L4 | 开放式长期自主能力 | 不在 1.0 承诺范围内 | 适配（非目标） |
| AGI | 通用智能能力 | 非协议版本目标 | 不在范围内 |

**L3 是 1.0 的上限。** L4 和 AGI 仅为界定承诺边界而声明，不列入计划。

## 发布路径

0.1 → 1.0 里程碑是交付时间线。每个发布版本标记其所推进的层和目标的自主级别。

| 发布版本 | 推进内容 | 目标级别 | 目标 |
| --- | --- | --- | --- |
| **0.1 Transition** | 事件 | L0–L1（文档化） | 建立 Harmovela 标识并文档化 L1 策略表面，不改变在线行为。 |
| **0.2 Core Stabilization** | 协调 | L1（冻结） | 冻结 L0–L1 协调核心：信封、会话、订阅、任务生命周期、错误、关联、版本协商、声明的交付语义。 |
| **0.3 Optional Profiles** | 协调 | L2（收紧） | 将委派/交接/升级/取消收紧为经一致性测试的配置文件；将持久化交付和安全拆分为可选配置文件。 |
| **0.4 Beta** | 协调 | L2（可互操作） | 通过两个独立实现的公开一致性矩阵验证 L2 多智能体协调。 |
| **0.5 Adaptation Preview** | 适配 | L3（规格化且可测试） | 定义反馈/结果关联和协议级预算、审计和授权语义；发布 L3 配置文件和跨语言一致性 fixtures。 |
| **0.9 Release Candidate** | 适配 | L3（已验证） | 通过可复现的 RC fixtures、兼容性矩阵、治理/安全/注册流程以及外部 L3 自主能力试点，在不扩展功能的情况下验证完整的 0.5 L3 语义。 |
| **1.0** | 适配 | L3（稳定） | 发布稳定的 L3 协调语义，并文档化声明 L4 和 AGI 为非目标的边界。 |

层文档提供各版本的入口和出口门禁。

## 近期优先事项

当前前沿是 **[协调层](roadmap/coordination-layer.md)**。在 0.1 标识迁移完成、L1 策略表面已文档化、委派语义已收紧的情况下，近期优先事项为：

1. **0.9 RC 验证** —— 通过可复现的 RC fixtures、兼容性矩阵、治理/安全/注册流程以及外部 L3 自主能力试点，在不扩展功能的情况下验证完整的 0.5 Adaptation Preview L3 语义。
2. **1.0 发布准备** —— 发布稳定的 L3 协调语义并文档化声明 L4 和 AGI 为非目标的边界。

## 层文档

层文档按其启用能力对历史工作进行分类，而非按最初交付的时序阶段。

- [事件层](roadmap/event-layer.md) — 基础层；吸收已完成的 Phase 0–8；稳定 (L0)。
- [协调层](roadmap/coordination-layer.md) — 活跃工作；委派和共享事实语义；发布版本 0.2–0.4 (L1–L2)。
- [适配层](roadmap/adaptation-layer.md) — 未来；反馈和受治理的产品化自主能力；发布版本 0.5–1.0 (L3)，L4/AGI 为非目标。
